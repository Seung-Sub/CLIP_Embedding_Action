# Q1. "g가 만드는 공간"은 괜찮은가?

## 1. 코드에서 확정되는 사실 (통화의 정체)

1. **phase2의 지도 타깃은 raw Δz가 아니다.** flow 분기: `lat_target = ae.g(cf, zc)`
   (train_phase2.py:666–667), 회귀 분기: `lat_target = ae.g(chunk_fut, z_cur)` (policy.py:171).
   즉 정책 f가 수송해 도착하는 곳은 **g의 출력 다양체**다. raw Δz(`z_next − z_cur`)는
   `wm` 손실로만 등장하고 **가중치 0** (전 canonical config `loss.wm: 0.0`).
2. **g와 Δz의 연결은 phase1 align 손실 하나뿐** — `MSE + 0.5(1−cos)` (networks.py:241–243),
   가중 0.5. 이는 소프트 제약이지 등식이 아니다 (실측 이탈도 §2).
3. **h는 두 분포를 모두 학습한다**: recon `h(Δz, z_t) ≈ a` (true Δz, w=0.5) + cycle
   `h(g(a,z_t), z_t) ≈ a` (g-출력, w=0.25) (networks.py:257–260). → **h의 입력 인터페이스는
   raw Δz 좌표계를 이미 수용한다.** 이 사실이 C3 방어(§3)와 R-Δ 셀(§5)의 지렛대다.
4. **추론 경로**: `f(tokens) → ζ̂ → h(ζ̂, z_cur) → a`. ζ̂의 학습 분포는 g-다양체 근방이므로
   추론 통화는 ζ ∈ g-space다. cycle 손실이 h를 g-다양체 위에서 훈련시켜 train/test 정합을
   맞추고 있다 (DESIGN_pipeline_rethink §3.2의 재확인).
5. **x0도 g-공간**: flow source=past → x0 = `g(A_past, z_prev)` 토큰. 즉 수송의 출발·도착이
   모두 g-공간이고, frozen 공간 원소(z_prev, z_cur, lang)는 조건으로만 쓰인다.

**따라서 정확한 서술**: *"frozen VL 공간의 변위 Δz를 통화로 쓴다"는 논문 문장은 현 구현에서
"Δz에 소프트 정렬된 학습 인코더 g의 출력 ζ를 통화로 쓴다"로 읽어야 한다.* 이 간극이
Q1의 실체이고, 그 크기는 아래 실측으로 이미 정량화되어 있다.

### 2.2 종합 판독 — "이탈"의 성격

- **ζ는 Δz의 손실 압축이다**: 방향은 대체로 보존(cos 0.7), 크기는 25% 수축, 고랭크 성분
  (369차원 중 ~250차원 분량의 구조)은 버려진다. PWCCA 0.32는 "정준방향으로 보면 절반도
  안 겹친다"는 뜻이므로, **"frozen 공간을 직접 쓴다"는 강한 주장에는 현 실측이 불리하다.**
- **그러나 버려진 것이 의미인가? — 실측은 반대를 가리킨다.** W4 v2-1 probe에서 운동학
  의미(축·방향·크기)의 선형 판별력은 **g가 dz보다 전 항목 높다**. 즉 g는 "행동이 만든
  변위 성분"을 농축하고, 소실분은 주로 (i) 크기 스케일 (ii) 행동과 무관한 고랭크 잔여
  (렌더/텍스처/조명 요동)로 추정된다 — 단 이 추정의 직접 증거는 아직 없다(§2.3 M2가 검정).
- **정책 수준의 추가 이탈**: lat_cos 0.2는 충격적으로 낮지만 SR 85–97.5와 공존한다.
  콜리그 eff-rank(∂h/∂ζ ≈ 5.5/1024)와 정합 — **h가 읽는 부분공간은 극소수 축**이라
  고차원 cos는 성능 지표로서 거의 무의미하다. 이는 "ζ 충실도"를 잴 때 전차원 cos가 아니라
  **h-가중(Jacobian-가중) 부분공간에서의 충실도**를 재야 한다는 방법론적 교훈이다.

## 3. (b) C3 semantic-interchange와의 충돌 — 얼마나 심각한가

### 3.1 문제의 정확한 형태

C3 헤드라인(LIT §3.3-①)은 `Δz_text = E_text(goal) − E_text(now)`를 "Δz 슬롯"에 주입한다.
그런데 추론 시 Δz 슬롯이라 부를 수 있는 곳은 두 군데고 좌표계가 다르다:

- **슬롯 A — 정책 출력 ζ̂ 자리 (h 입력)**: 학습 분포 = g-다양체(cycle) **∪ true Δz(recon)**.
- **슬롯 B — 정책 내부 (flow 타깃/토큰)**: 순수 g-공간 + 조건 토큰은 frozen 공간.

우려("텍스트 델타가 틀린 좌표계에 산다")는 슬롯 B에 주입할 때 전면적으로 성립한다 —
Δz_text는 g가 만든 적 없는 벡터이고, 크기 25% 수축·저랭크 압축이라는 g-특유의 통계를
갖지 않는다. **그러나 슬롯 A는 다르다**: h는 recon 항(w=0.5, cycle의 2배)으로 **true Δz를
5만 스텝 이상 직접 소비하며 학습**했다. 따라서 raw-Δz 좌표계 벡터는 h에게 in-distribution
이다. **C3의 올바른 주입 지점은 h이며, 그 경우 "g-공간 통화" 문제는 C3를 훼손하지 않는다.**

남는 정량적 위험 3건 (전부 측정 가능):

1. **텍스트-델타 ↔ 이미지-델타 방향 정렬 자체**. 실측된 CKA(Δz, 모션문장) 0.026–0.035,
   mutual-kNN ≤2%는 나쁜 신호처럼 보이지만 **절대 텍스트 임베딩** 대비 측정치다.
   modality gap [V 2203.02053]은 모달리티별 상수 원뿔 오프셋이 지배 — **델타끼리의 비교
   (Δz_text vs Δz_img)에서는 상수 오프셋이 소거**되므로 별도 측정이 필요하다(미실측).
   경고적 선례: SIMAT [V 2112.03162] "vanilla CLIP delta transport는 약하고 가벼운
   파인튜닝으로 크게 개선", PC-CLIP [V 2409.09721] = 정확히 이 정렬을 학습하는 방법.
   → **신규 측정 M6**: 과제별 상태문장 쌍으로 CKA/cos(Δz_img 과제평균, ΔE_text) 행렬 —
   matched 대각 우세 여부. CPU 반나절, C3 go/no-go의 1차 게이트.
2. **스케일**: ‖Δz_text‖는 과제 전체의 의미 이동, h가 기대하는 Δz는 0.8s 청크 변위.
   보정 스칼라 s(청크 노름 중앙값 매칭) 필수 — §3.4 T-0에 포함.
3. **concat 기질의 반쪽 문제**: DINOv3 블록[1024:2048]은 텍스트 타워가 없어 Δz_text가
   채울 수 없다. 옵션 = 0-채움 / ridge r(z_t) 예측 채움 / **large256 단독 기질(1024)에서
   먼저 실험**(권장 — S1b-noalign이 보인 언어 최우수 지대이기도 함).

### 3.2 "g를 통과시키는" 방식은 성립하는가 — 타입 검사

g의 시그니처는 `g(action_chunk, z_t)`다. **Δz_text는 액션이 아니므로 g에 넣을 수 없다** —
"텍스트 델타를 g로 사상"은 타입 오류로 기각. 유일하게 유의미한 "g-경유" 해석은 역방향,
즉 **h를 경유**하는 것: `a = h(s·Δz_text, z_t)`. h는 (변위→액션) 역모델이고 true Δz로
학습됐으므로 이것이 문법에 맞는 유일한 zero-shot 경로다. (그다음 단계로 h가 산출한 a를
g에 되돌려 `ζ_text = g(h(s·Δz_text, z_t), z_t)`로 "텍스트 델타의 g-좌표 사영"을 얻는
왕복 트릭도 가능 — 슬롯 B 주입이 꼭 필요해질 때의 보정기 후보. 학습 0.)

### 3.3 수정 옵션 서열 (C3 관점)

| 순위 | 옵션 | 내용 | 비용 | 평가 |
|---|---|---|---|---|

### 3.4 T-0 스펙 — "존재하는 가장 싼 C3 실험" (사전등록, 오프라인 우선)

- **가설**: h는 recon 항으로 true Δz를 학습했으므로, 스케일만 보정된 텍스트 변위
  s·Δz_text에서 **과제 방향과 일치하는 액션**을 zero-shot으로 복호할 수 있다.
- **기질**: SigLIP2-large256 단독(1024d, dim_text=dim — anchor.py:108 확인). concat 제외(§3.1-3).
- **입력 구성**: 과제별 상태문장 쌍 8–10개 (예: "a black bowl on the table" → "a black bowl
  on the plate"; W4v3-P2 SpLiCE 관계 어구 24종 재사용). `Δz_text = E_text(goal) − E_text(now)`,
  `s = median‖Δz_train‖ / ‖Δz_text‖` (val 통계, 과제 공통 1스칼라).
- **오프라인 게이트 (롤아웃 0회, GPU <0.5h)**:
  - G-T0-a: 데모 val 상태 z_t들에서 `â = h(s·Δz_text, z_t)` vs 그 상태의 실제 데모 액션 —
    **EE 이동 방향 cos**(지배축 부호 일치율)과 그리퍼 부호. 성공 기준: 방향 cos 분포
    중앙값 > 0 + 셔플 문장 대비 유의 우위(문장-셔플 대조군 필수 — 상태 지름길 차단).
  - G-T0-b: 대조 민감도 — goal 문장을 wrong 과제 문장으로 스왑 시 â 변화량(cos 거리)이
    same-문장 노이즈 대비 유의하게 클 것 (C3 ⑤ blank/paraphrase 프로토콜의 오프라인판).
  - G-T0-c: M6 행렬(§3.1-1)의 matched-대각 우세 — a·b와 독립적 기하 증거.
- **통과 시에만** 폐루프: 재계획마다 ζ̂ 대신 s·Δz_text를 h에 주입(또는 ζ̂와 혼합 β 스윕)한
  zero-demo 롤아웃 5태스크 × 10롤 스크리닝. 예상 실패 모드도 사전 기록: (i) 방향은 맞으나
  s가 상태 의존이어야 함(고정 s의 한계) → per-state s 회귀 1개 허용, (ii) SIMAT형 실패 —
  델타 정렬 자체가 약함 → PC-CLIP식 보정기(§3.3-5)로 승격하되 "보정기 필요"를 정직 보고.
- **정보 가치**: 성공 = C3 ①의 최소 실증(LIBERO-CF 좌표계 진입 전 내부 증거). 실패 =
  보정기 필요성의 정량 근거. 어느 쪽도 논문 §C3에 기록됨. **비용이 0에 가까우므로
  기대값 논쟁 자체가 불필요** — week-0 목록(pipeline_rethink §10)에 추가 권고.

## 6. R-Δ 셀 — 사전등록 스펙 (RAW-Δz grounding)

- **질문**: 통화를 g-출력에서 raw frozen 변위로 바꾸면 폐루프가 유지되는가?
  유지되면 → "frozen 공간 직접 사용" 주장이 문자 그대로 참 + C3 좌표계 문제 소멸.
  하락하면 → g의 디노이징/압축이 하중 지지임이 **구성적으로** 입증 (§5 가설 1·2).
- **설계 (정책·h 구조 무변경 — Q2와의 직교성이 이 셀을 싸게 만든다)**:
  - flow 타깃: `lat_target = zn − zc` (train_phase2.py에 이미 wm_target으로 존재하는 텐서 —
    타깃 텐서 스왑이 구현의 전부). `x0_std`는 Δz 분포로 재설정.
  - a_emb/x0 토큰: `g(A_past, z_prev)` → **`Δz_past = zc − zp`로 교체** (파라미터 0, 캐시
    기존재). past-결합 관례(캠페인 승자) 유지 — 출발·도착이 모두 raw-Δz 좌표계로 통일.
  - 디코더 arm 2개: **arm-1** h 그대로(recon이 true Δz로 학습돼 있어 zero-shot 호환 —
    단 ζ̂_raw의 정책 오차 분포는 cycle이 커버하던 역할이라 미보장) / **arm-2** h만 재적합:
    recon + **입력 노이즈 항** `h(Δz + σε, z_t) ≈ a` (σ = phase2 val 잔차 실측치) —
    cycle이 하던 "test-time ζ̂ 분포 커버"를 노이즈/다양체 정규화가 대신한다. cycle 제거.
  - **PCA-k 자매팔** (pipeline_rethink §6 승계): 타깃 = P_k Δz (k=32), flow_dim=k.
    §5 가설 2의 분리 검정. M5 게이트(상위 32축 분산비 ≥95%) 통과 시에만.
- **오프라인 게이트 (롤아웃 전, 순서 고정)**:
  1. G-R-1: phase2 val act-R² ≥ base −0.01 (base: large256 0.655 / concat 0.683).
  2. G-R-2: `h(ζ̂_raw, z_cur)` 디코딩 건강 — 그리퍼 acc ≥ 92%, MAE 비열화.
  3. G-R-3: ζ̂_raw vs Δz cos ≥ 현 lat_cos 수준(0.20) — 미달이어도 M3(h-가중 사영)로 재판정.
  4. (arm-2만) recon R² 비열화 확인 후 진입.
- **폐루프**: 스크리닝 10태스크 × 20롤 correct+wrong, paired vs 동일 기질 base.
  판정 = 비열등(CI 하한 > −5pp; P-C와 동일하게 **우월 검정 아님을 명시**) AND
  correct−wrong ≥ +70pp 공동기준.
- **비용**: arm-1 = phase1 재학습 0 + phase2 1.5h + probe + 롤아웃 6h ≈ **0.5 GPU-일**.
  arm-2 = +phase1-h 재적합 1h. PCA-k 팔 +1.5h. 합계 ≤ 1.5 GPU-일 (P-C JOINT-ζA ~1주의 1/4).
- **예측 (사전 기록)**:
  - H-g(g-공간 유익) 하: arm-1 SR 70–85 (h 분포 불일치 + 노이즈 통화), arm-2 부분 회복
    80–90, PCA-k ≈ base. → 서사: "frozen 변위는 **압축·정제되어야** 통화가 된다."
  - H-raw(충실도 승) 하: arm-2 ≥ base(비열등), 언어축 유지·개선(통화가 frozen에 더
    가까워지므로 C3·SpLiCE 해석이 전부 직통). → 서사: "정말로 frozen 변위 그 자체가 통화."
  - 어느 쪽이든 §5 표의 반증 조건들이 닫히고, C3 주입 좌표계가 확정된다.

# Q2. FlowPolicy는 가장 적합한 구조인가

## 7. 현 모델의 정밀 특성화 (코드 실측)

- **구조**: 어텐션 제로. `ctx` = 토큰 5–6개(각 1024–2048d) flatten-concat → LN → Linear(→1536)
  → pre-LN 잔차 FFN ×2 (ctx_layers). 속도장 v = Linear(flow_dim+1536+128) → 잔차 FFN ×4
  (layers) → Linear(→flow_dim). Euler 6스텝. `heads` 인자는 미사용(사어).
- **토큰**: [z_prev, z_cur, g(A_past), lang, wrist] = 5 (+obs/grid 시 +K). 위치 규약 고정
  (A_EMB_IDX=2). 조건화는 flatten이므로 "슬롯 = 위치 인코딩" (patch 설계 문서 §2.3과 정합).
- **파라미터 실측** (본 세션 계산): 768d/5tok **124.2M** · 1024d/5tok **126.9M** (회수 grid
  json의 n_params=126,910,848과 일치) · 2048d/5tok **137.9M** (ctx 53.5M + v-net 84.4M).
  "작은 네트워크"는 **토큰/구조 관점**에서만 참이다 — 파라미터로는 π0 액션 전문가급 규모.
- **x0=past + source_noise 0.1은 train 전용** (policy.py:117 `self.training` 가드) →
  **추론은 완전 결정론**: ζ̂ = ODE(g(A_past); ctx). 노이즈 샘플링 자체가 없다.
- **학습 곡선 (회수 wandb, a6000)**: train/total 0.10–0.24 vs val/total 0.37–0.39 —
  **2–4× 과적합 갭**. val/lat(CFM)은 0.006–0.008로 이미 바닥, val/act(0.37–0.39)가 전부.
  즉 21.7k 샘플 레짐에서 127M 정책은 용량 부족이 아니라 **용량 과잉 + 조기 종료로 관리**
  되는 상태다. (early_stop_patience 20, 50ep 내 미발동 — best-val 스냅샷이 사실상 정규화.)
- **flow vs 회귀의 격차는 샘플링이 아니라 학습 목적함수**: MLPConcat 회귀 35.6% vs flow
  87%인데 추론은 둘 다 결정론 — CFM의 반복 정제 + past-결합(x0가 정답 근방에서 출발)이
  회귀 대비 우월한 것이지, 확률성이 이득의 원천이 아니다. (확률화하면 오히려 33–37%.)

과하지도 모자라지도 않는다. 규모 문헌(π0/GR00T)은 **문제 형상이 다르다**(멀티태스크·

## 9. 통합 — Q1×Q2 상호작용과 큐 랭킹

### 9.1 상호작용

- **R-Δ × FlowPolicy = 무간섭**: flow_dim=latent_dim 그대로, 타깃·x0 텐서만 교체.
  정책 구조 재설계 없이 통화 실험이 가능하다는 것이 Q2 "KEEP" 판정의 실질 배당.
- **R-Δ 성공 시 T-0/C3가 직통**: ζ̂와 Δz_text가 같은 좌표계 — 주입 슬롯 A/B 구분 소멸.
- **patch 조건화(P-B) × Q2**: ctx flatten은 K=8까지 무리 없음(§8-ii). P-B가 이기면
  그때 토큰 수 기준으로 (i) 재검 조건이 열린다 — 순서 의존성 명시.
- **PCA-k × Q2**: flow_dim=32로 축소 시 v-net 파라미터 대폭 감소 — (v) 축소 스윕과
  자연 결합 (한 셀에서 두 질문 동시 응답).

### 9.3 논문 서사에의 함의 (한 문단)

현 실측으로 정직하게 쓸 수 있는 문장은 *"우리 통화는 frozen 변위 Δz에 소프트 정렬된
(cos 0.65–0.69, 노름비 0.75) 저랭크 인코딩 ζ이며, 운동학 의미는 이 인코딩에서 보존·농축
된다(probe selectivity g>dz)"*이다. R-Δ가 비열등으로 나오면 "frozen 변위 그 자체"로 문장이
강해지고 C3 좌표계 문제가 소멸한다. R-Δ가 열등으로 나오면 "frozen 변위는 압축·정제를
거쳐야 통화가 된다"가 되고 g의 역할이 **기여**로 승격된다 — 어느 쪽도 논문을 해치지 않으며,
지금처럼 측정 없이 두는 것만이 리뷰어 공격면(“직접 쓴다더니 학습 공간이네”)으로 남는다.
