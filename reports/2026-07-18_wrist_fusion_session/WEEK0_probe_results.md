## 0. 실행 환경 주의 (GPU 아님 — CPU)

원격 박스 컨테이너는 **신규 프로세스의 CUDA 접근이 전면 불가** 상태였다(지시된 GPU 8/9 포함
전체): `nvidia-smi` NVML "Unknown Error", 모든 python env에서 `torch.cuda.is_available()=False`,
`/dev/nvidiactl` 읽기 자체가 거부(cgroup v2 eBPF device filter). 컨테이너 내부 권한
(CapEff 00000000a80425fb)으로는 복구 불가 — 호스트 관리자의 컨테이너 재기동/디바이스 룰 복원
필요(0842fb1 커밋의 "박스 GL-stack admin" 이슈와 같은 뿌리, 이번엔 GL을 넘어 CUDA 전체).
**따라서 전 프로브를 CPU로 실행**했다. 모든 프로브는 소형 MLP forward + ridge 회귀뿐이라
수치는 device 무관(캐시 임베딩 재사용, 인코더 미로드 — `DummyAnchor(cache_key, dim)`).
결과 JSON `_meta.device`에 명기.

**재현 무결성(sanity)**: 모든 팔에서 split을 학습 코드와 동일하게 재현
(`RandomState(seed).permutation`, val=20%): phase1 쌍 22014/5243, phase2 dual val 5502가
ckpt 기록과 정확히 일치하고, 재계산한 align cos·dec R²가 ckpt metrics와 소수 4자리까지 일치
(예: concat +0.6684/+0.7493). 프로브 수치는 학습 당시 분포 그대로에서 측정된 것.

**기질(ckpt)**: S1 승자 팔 — `phase1_libero_fobsfusion_concat.pt`(dualconcat 2048) 및
`phase1_libero_fobsfusion_avg.pt`(dualfusion 1024) 중심, 비교축으로
`phase1_libero_siglip2_large256.pt`(1024)·`phase1_libero.pt`(CLIP 768). wrist는
`phase1/phase2_libero_dualstream_wrist.pt`.

## 1. P-zg1 — 액션-셔플 (g가 액션을 실제로 읽는가)

**설정**: held-out val 쌍에서 배치(512) 내 액션청크 무작위 permute 후
g(a_π(i), z_i) vs Δz_i 의 align cos·MSE를 원본 g(a_i, z_i)와 비교. 셔플 시드 5개 평균.
**적신호 기준(사전등록)**: 셔플 cos ≥ 0.5×원본 → 상태 지름길.

| 기질 | 원본 cos | 셔플 cos | 비율 | 적신호 |
|---|---|---|---|---|
| concat 2048 | +0.6684 | +0.4703 (±0.001) | **0.704** | **RED** |
| avg 1024 | +0.6702 | +0.4532 | **0.676** | **RED** |
| CLIP 768 (참조) | +0.6544 | +0.4234 | **0.647** | **RED** |
| large256 1024 (참조) | +0.6470 | +0.4426 | **0.684** | **RED** |

**판정: 적신호 성립(전 기질).** align cos의 약 65–70%가 액션을 몰라도(상태만으로) 나온다.
단 완전 붕괴는 아니다: 액션이 기여하는 나머지 ~30%가 실재하고, 셔플 시 g의 pooled R²는
0.476→0.211로 절반 이하로 떨어진다(액션 정보가 실제로 통과함 — D4 다봉성·ARM-AE 대조와 정합).
**함의**: "align cos가 높다 = 액션 접지"라는 서사는 그대로는 방어 불가. align 지표에서
상태-예측가능 성분을 빼고 보고해야 함(§2의 잔차화 방향).

## 2. P-zg3 — 상태-단독 상한 (RidgeCV z_t→Δz)

**설정**: `f2_dense_probe.py` 패턴(RidgeCV, α∈{0.1…1e4}, train 22014 → held-out 5243),
z_t만으로 Δz 예측한 pooled R²·cos을 g의 align R²·cos과 비교.
**적신호 기준**: g align R² ≈ ridge R² → g가 상태예보기.

| 기질 | ridge(z_t→Δz) R² | ridge cos | g align R² | g cos | gap(g−ridge) | 적신호 |
|---|---|---|---|---|---|---|
| concat | **+0.5400** (α100) | **+0.7139** | +0.4763 | +0.6684 | **−0.064** | **RED** |
| avg | +0.5384 | +0.7112 | +0.4753 | +0.6702 | **−0.063** | **RED** |
| CLIP (참조) | +0.4896 | +0.6727 | +0.4637 | +0.6544 | **−0.026** | **RED** |
| large256 (참조) | +0.4638 | +0.6671 | +0.4523 | +0.6470 | **−0.012** | **RED(경계)** |

**판정: 적신호 성립 — 예상보다 강하게.** 상태-단독 선형 ridge가 g보다 pooled R²로 +0.064,
cos로도 +0.045 **더 높다**. 즉 현행 align 지표(cos/MSE/R²) 위에서는 "액션 없는 상태예보기"가
학습된 g를 능가한다. g가 상태예보기라는 뜻은 아니고(P-zg1에서 액션 기여 실증), **align
지표가 상태-예측가능 성분에 지배**된다는 뜻. Δz 분산의 ~54%는 z_t의 선형 함수다
(데모가 전문가 정책이라 a≈π*(z_t)인 BC 지름길 구조가 실측으로 확인됨).
부수 발견: 상태-지배 gap이 융합 기질(concat/avg −0.063)에서 단일 앵커(clip −0.026,
large256 −0.012)보다 크다 — 융합 z_t가 Δz를 더 잘 예보(공간정보↑)하므로, 융합 기질일수록
잔차화의 가치가 커진다.
**함의**: 논문 방어용으로 "align cos 0.67" 단독 인용은 위험. innovation-grounding
(align 타깃을 Δz − r(z_t)로 잔차화, §1.4 대안 1)이 이제 "대안"이 아니라 **1순위 후속 셀**
자격 — 사전등록대로 적신호 발동 조건 충족.

## 3. P-zg4 — 통화 공동화 프록시 (I(ζ; z_t))

**설정**: (a) RidgeCV ζ→z_t held-out R² (ζ=g(a,z_t), 대조군 Δz(GT)→z_t);
(b) 디코드 절제 — h 입력 zero 2×2.
**적신호 기준**: h(0,z_t) R²가 h(ζ,z_t)에 근접 → 통화 공동화.

concat 실측:

| 프로브 | R² |
|---|---|
| (a) ridge ζ→z_t | **+0.9012** |
| (a-ctrl) ridge Δz→z_t | +0.5316 |
| (b) h(Δz, z_t) [dec] | +0.7493 |
| (b) h(ζ̂, z_t) [cycle] | +0.8993 |
| (b) **h(0, z_t)** | **+0.2553** |
| (b) h(Δz, 0) | +0.1604 |
| (b) h(ζ̂, 0) | −1.074 |
| (b) h(0, 0) | −0.001 |

전 기질 교차 확인 (JSON `w0_p1_{arm}.json`):

| 기질 | ζ→z_t R² (ctrl Δz→z_t) | h(ζ̂,z_t) | h(0,z_t) | zg4b 적신호 |
|---|---|---|---|---|
| concat | 0.901 (0.532) | +0.899 | +0.255 | OK |
| avg | 0.931 (0.505) | +0.889 | +0.258 | OK |
| CLIP | 0.965 (0.454) | +0.880 | +0.220 | OK |
| large256 | 0.951 (0.468) | +0.866 | +0.201 | OK |

**판정: (b) 적신호 불성립 — h는 통화를 실제로 사용.** h(0,z_t)=0.255 ≪ h(ζ̂,z_t)=0.899.
상태만으로 액션 R² 0.26은 D4 결정론 상한(R²≈0.59)보다도 낮고, ζ 채널이 디코딩 기여의
대부분을 지지함이 확인된다 — "통화 공동화" 시나리오 기각.
**(a)는 경고 신호**: ζ에서 z_t가 R² 0.90으로 선형 복원된다 — 타깃 Δz 자체가 갖는
상태정보(0.53)보다 훨씬 많다. 즉 g는 임무에 필요한 것 이상으로 z_t를 ζ에 실어나른다
(concat 2048에는 병목이 없어 자유 통과). 폐루프 해악의 직접 증거는 아니나(phase2가
z_cur를 어차피 보므로 정보 중복일 뿐), C3 해석가능성 주장("ζ≈Δz")의 정밀도를 깎는 요소.
**함의**: 적신호 2/3 구도(P-zg1·zg3 RED, zg4b OK) — "ζ는 액션 통화이되, align 손실과
ζ 내용의 절반 이상이 상태 성분"으로 서사를 정정해야 함. 잔차화 타깃이 (a)도 함께 완화할
것으로 예측(등재).

## 4. h Jacobian eff-rank (∂h/∂ζ) — §6

**설정**: val 256샘플에서 ∂h/∂ζ (112×latent) SVD → participation ratio(PR),
rank@90% energy. 평가점 2종: (Δz_GT, z_t) 및 (ζ̂=g(a,z_t), z_t). 콜리그 참조 ≈5.5/1024.

|---|---|---|---|---|---|
| concat | 2048 | 4.96±0.72 | 5.8 | 5.06±0.67 | 5.9 |
| avg | 1024 | 4.57±0.72 | 5.4 | 4.66±0.66 | 5.5 |
| large256 | 1024 | 3.93±0.67 | 4.9 | 3.98±0.64 | 4.8 |

**판정: 콜리그 재현 — 동급(PR 4–5/전체차원), 기질 불문.** h가 실제로 읽는 유효 방향은
~5개(≤0.5%)뿐이고 latent 폭(768→2048)과 거의 무관하게 일정하다. 출력(112차원 액션청크)
조차 상위 ~6 SV가 에너지 90%를 담는다.
**함의**: §6 처방 그대로 — "h 확장"은 무의미, **PCA-k(k=32) 통화 압축 셀**이 다음 유효
실험. 압축은 P-zg4(a)의 상태 중복 완화와도 방향이 같다(저분산 상태축 절단).

## 6. ζ_wrist zero-ablation + ego-motion — DESIGN_wrist_v2 Stage 0

**설정**: dual ckpt(phase1+phase2), phase2 seed2 val 5502샘플 — ckpt와 동일 split·산식
재현(full R² 0.6626 = ckpt 0.6626 정확 일치). run_dual 평가 경로 동형으로 ζ̂ 생성 후
조건별 디코드 액션 R². **게이트(사전등록)**: ΔR²(A−B) < 0.005 kill / ≥ 0.02 GO.

| 조건 | 액션 R² | gripper-dim R² |
|---|---|---|
| A full: h(ζ̂, z) | **+0.6626** | +0.795 |
| B ζ̂_wrist=0 | +0.4832 | +0.656 |
| C ζ̂_main=0 | +0.5780 | +0.765 |
| D z_wrist 셔플(상태만) | +0.6532 | +0.774 |
| A′ oracle GT Δz | +0.7757 | +0.900 |
| B′ oracle wrist=0 | +0.6748 | +0.778 |
| C′ oracle main=0 | +0.5810 | +0.890 |

- **ΔR²(A−B) = +0.179 → GO** (게이트 0.02의 9배). ζ_wrist는 오프라인에서 죽은 채널이
  아니라 **실질 기여 채널**. oracle에서도 Δ +0.101. wrist 단독(C 0.578)이 main 단독
  (C′와 비교) 못지않음 — dual h는 wrist 변위에 상당히 의존하도록 학습됨.
- D(z_wrist 상태 셔플)는 −0.009뿐 → 기여는 상태조건이 아니라 **변위 ζ_wrist 자체**.
- oracle gripper-dim: main=0에서도 0.890 유지 — 파지 채널은 wrist 변위만으로 거의 완전
  디코딩 가능(G3a 방향의 방증).
- **함의**: Phase-B 폐루프 널은 "wrist 변위에 정보 없음"이 아니라 실행/스케일/커버리지
  문제였을 개연성 강화 — §9 재개 조건("zero-ablation 기여>0") **충족**, Stage 1
  (표준화+블록 x0_std 재판) 진입 자격 성립.

**ego-motion 프로브(E1)**: RidgeCV 액션청크(112, 정규화)→Δz_wrist held-out R²
(phase1 seed0 split, train 22014/val 5243). 게이트: R²≥0.5 → ego 지배 / <0.3 → 근거 약화.

| 팔 | R² |
|---|---|
| E1 액션→Δz_wrist | **+0.177** (α100) |
| ctrl 액션→Δz_main | +0.115 (α1e3) |

**판정: ego-motion 지배 가설 기각(선형 기준).** 예측 P0-2("E1 R²≥0.5") **반증**.
Δz_wrist 분산의 ~18%만 액션청크의 선형 함수 — "wrist 변위 = 액션으로 자명한 optic flow"
서사는 이 검정에선 성립하지 않는다(main 대비 +0.06 높아 방향성만 어렴풋).
**함의**: §4a(EE-frame 좌표 재정렬)의 선행 게이트 G3b 미통과 → 4a 우선순위 강등,
**§4c(wrist 국소 기하 조건 토큰)가 1순위 유지**. 주의: 선형 프로브 한계 — 비선형(MLP)
판·E2(proprio)·R1(잔차)·국면분해(프로브 4)는 미실행이므로 G3b 판정은 "약한 근거" 등재
(G3a/G3c의 완전 판정은 후속).

## 7. C0 재확인 — decoder_state_cond=False "거의 무손실" 주장

**설정**: C0 재학습 ckpt는 원격에도 **부재**(checkpoints/·grid/ 확인) — 1차 아티팩트
UNTRACED 유지. 대신 **근사 절제**(학습된 h에 z_t=0 주입, §3 표의 h(Δz,0)·h(ζ̂,0) 행)를
실행하고 근사임을 명기한다.

- 근사 절제 h(Δz,0) R²: concat **+0.160** / avg +0.284 / CLIP +0.225 / large256 +0.117
  (full 대비 −0.45 ~ −0.59), h(ζ̂,0)은 붕괴(concat −1.07) — **근사 절제에서는 "거의
  무손실"이 전혀 재현되지 않음.**
- 단, 이것은 C0 주장의 반증이 **아니다**: z_t=0 주입은 LayerNorm 입력분포 이탈을 포함한
  하한 근사이고, C0의 원 주장은 "state-cond 없이 **재학습한** h"에 대한 것. P-zg4(a)에서
  ζ가 z_t를 R² 0.90으로 담고 있으므로, 재학습 h는 ζ에서 상태를 회수해 실제로 거의
  무손실일 수 있다(정합적 설명).
- **판정: C0 "거의 무손실"은 UNVERIFIED 유지** — networks.py docstring의 해당 문구는
  1차 근거 회수 또는 재학습 1런(GPU 복구 후, `decoder_state_cond=false` 1줄) 전까지
  인용 금지 권고.
