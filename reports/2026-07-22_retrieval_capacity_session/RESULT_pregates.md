## 1. G-D1 — 의도판독 데모 사전게이트: **FAIL** (재적합 포함 2단 판정)

**프로토콜** (사전등록대로): matchedbase phase2 (seed2, `phase2_libero_large256_matchedbase.pt`) + phase1 large256.
held-out 100 에피소드(seed2 split = R-0/phase2 동일), 데이터셋 조건 토큰 [z_prev, z_cur, g(A_past), lang, z_wrist]로
오프라인 정책 forward, 결정적 x0=past(eval 모드), 리플랜 격자 t = 16, 24, 32, …(exec-horizon 8 케이던스).
라벨 = R-0 뱅크 세그먼트 중 미래 창 [t, t+16)과 최대 중첩. 판독 = P_action(ζ̂) vs canonical-8 P_text 최근접.
리플랜 포인트 1,224개 전부 라벨됨 (순수-창 589 / 경계-창 635).

**결과**:

**혼동 행렬 (원 어댑터, ζ̂, 행=GT / 열=예측; 클래스 순서 approach·grasp·back·fwd·left·right·place·release)**:
approach의 60%(186/310)가 place로, left의 45%가 grasp로 빨려감; back·right는 전멸(표본 4·13).
place(0.63)·fwd(0.54)·grasp(0.50)만 부분 생존 — "place/grasp/fwd 3개 흡인자" 패턴.

**재적합 후 혼동 (ζ̂-재적합 어댑터)**: approach 0.94 / grasp 0.80 / place 0.86 / left 0.78 / fwd 0.76으로 회복.
잔여 누수는 두 곳에 집중 — ① **release→place 62/145** (시간상 인접한 하강-개방 국면의 경계 혼동),
② 극소표본 클래스(back n=4 → 0.0, right n=13 → 0.46). 방향 클래스 간 혼동은 크지 않음.

**실패 분해 (사전등록된 진단 질문: ζ̂ 노이즈인가, 혼동 패턴인가 → 답: 둘 다 아닌 "분포 이동 + 입도", 재적합 후엔 국소 혼동만)**:
1. **입도(세그먼트→청크) 비용**: 데이터셋 ζ조차 청크-급에선 0.972 → 0.657. 어댑터는 가변길이 전체 세그먼트 ζ로 학습됐고,
   0.8초 청크는 다른 분포다 (approach→place 혼동이 GT ζ에서도 대량 발생).
2. **ζ̂ 분포 이동**: cos(ζ̂, ζ_GT) = **0.188** (q05 0.026 / q95 0.358). 이는 하네스 버그가 아님 —
   학습 시 val lat_cos가 정확히 **0.203** (ckpt metrics). 이 정책의 ζ̂는 원래 g-타깃과 cos가 낮은 채로
   행동 디코딩만 정확한(act R² 0.655) 부류. correct/wrong 포인트 간 cos 차이 없음(0.195 vs 0.183) → "노이즈가 클수록 틀린다"가 아니라 **전역 분포 이동**.
3. **재적합(§3 사전등록 구제, `adapter_zetahat.pt`)**: train 에피소드 ζ̂ 4,838개로 동일 레시피 재학습 →
   held-out ζ̂ 0.786. 회복 큼(+0.42)이나 0.85 미달 → **§5 E2 조항 발동: 데모 강등**.
4. **부분 구제 수치 (데모 UI 마진 임계 관점, §3의 m<0.1 "불확실" 표기 설계)**:

   → "매 스텝 판독" 데모는 죽지만, "고마진 스텝만 자막" 판(커버리지 80%)은 0.85 선을 넘는다 — 강등된
   오프라인 그림의 캡션에 정직 공개 조건으로 수록 가능. 채택 여부는 PI 판단.
5. 클래스 불균형 주의: backward(train 5/heldout 4)·right(61/13) 표본 극소 — per-class 수치는 이 두 클래스에서 무의미.

## 2. R-D0 — W-D 사전 릿지: **GO (마진 +0.0027 — 아슬아슬)**

**프로토콜**: 타깃 = Zw_sig(t+16) − Zw_sig(t) (SigLIP2-large256 pooled wrist RAW 캐시 — 실존 확인:

|---|---|---|---|
| r_state [z_main ⊕ z_wrist] | 2048 | 0.5277 | — |
| **r_full [state ⊕ chunk112 ⊕ a_emb]** | 3184 | **0.5504** | **+0.0227** |
| state ⊕ chunk112 | 2160 | 0.5373 | +0.0096 |

- 마진이 0.0027로 임계 직상 — "액션 특징이 담는 초과 신호"는 실재하되 얇다. 초과분은 거의 전부

- 흥미 진단: DINO wrist 패치가 액션 특징보다 미래 손목 변위를 더 잘 예측(+0.0443) — "미래 손목 변위"의
  상당분이 현재 손목 시각 디테일로 설명됨.
- 본 게이트는 W-D의 **필요조건**만 통과시킨 것. 실행 순서(§4.2: W-A 확증 판정 후)와 G1/G2-D 게이트,
  그리고 PAPER v2 §5 E8의 "보류/킬" 권고(어느 기여도 W-D를 요구하지 않음)는 그대로 유효 —

## 4. 종합 — 무엇이 살고 죽고 기다리는가

- **E2 의도판독 데모**: **강등(사전등록 조항 발동)** — "오프라인 판독 그림"으로. 단 재설계 옵션 (a)+(b)
  (마진-게이팅 + place/release 병합 = 0.856@80% / 0.871)이 수치로 확보돼 있어 PI 재가 시 제한적 데모 부활 가능.
  ζ̂-재적합 어댑터(`adapter_zetahat.pt`)가 어느 판이든 필수 기반(원 어댑터는 ζ̂에 사용 불가, 0.363).
- **W-D (E8)**: 킬게이트 **생존 (대기)** — 마진 +0.0027의 아슬아슬한 통과. 착수는 여전히 W-A 확증 판정
  + PI의 E8 "보류/킬" 권고 해제에 종속. 이 게이트는 필요조건 통과일 뿐.
- **W-A′ (E6)**: 킬게이트 **생존 (대기→진행 자격)** — 기전 귀속의 절반("인코더-불문")을 오프라인에서 이미 획득.
  W-A 확증 종료 후 §4.2 분기대로 착수 (특히 확증이 "SR 승리·언어 미달"로 끝나면 결정 셀로 승격).

*실행 메모: 3게이트 전부 CPU-only 준수 (rollout_sim W-A 확증 2프로세스 무간섭 확인, GPU 차용 0회).
프로토콜 위임 해석 1건 기록: R-D0의 r_full "past-chunk features"는 과제 지시대로 [과거청크112 ⊕ a_emb]
(설계문서의 DINO-패치 변형은 진단팔로 병산, +0.0443 GO 방향 동일 — 판정 불변).*
