# 손목카메라(W-A) 스크리닝 결과 보고 (2026-07-20~21, kist_a6000_ss)

> 대상 독자: PI. 전문용어 최소화, 결론 우선.
> 이 문서는 W-A(WristPatch) 팔의 결과입니다. W-B/W-C 팔 결과는 2026-07-21 완료되어
> §6에 추가됐습니다 (판정: 둘 다 널 → W-A 확정실험 단독 집중).

## 0. 한 줄 결론

**손목카메라 패치 토큰(W-A)을 더한 정책이 기준선보다 평균 +5.5%p 높았고(87.0 → 92.5%),
이득은 설계가 예측한 "잡기(파지)가 어려운 태스크"에 집중됐다(서랍 태스크 +35%p).**
다만 20회 스크리닝의 통계 검정으로는 아직 "확실한 승리" 선을 넘지 못했고(신뢰구간이 0을 스침),
언어 순응도(정상−틀린 지시 격차)가 유보 구간에 걸렸다.
→ **사전등록 규칙(A4 유보 조항)에 따라 확정실험(50롤×3시드)으로 올려 재판정할 것을 권고.**
즉시 탈락(NO-GO) 조건에는 어느 것도 걸리지 않았다.

## 1. 이번 비교가 왜 특별한가 — 프로젝트 첫 "제대로 짝지은" 손목 비교

지금까지의 손목카메라 실험(Phase-B dual 등)은 기준선이 다른 주(週)·다른 코드 리비전에서
학습된 것이라 "손목 때문인지, 다른 차이 때문인지"를 분리할 수 없었다. 이번에는:

- **기준선(matchedbase)을 같은 날, 같은 코드, 같은 데이터 분할·시드로 재학습**했고
  (유일한 차이 = 손목 패치 토큰 4개의 유무),
- phase1(동결 인코더/디코더)은 두 팔이 **같은 파일을 공유**하며,
- 롤아웃도 **같은 초기상태 세트로 짝지어(paired)**, 두 GPU에서 **기준선↔실험팔을 교대로**
  같은 시간대에 돌려 서버 상태 차이까지 제거했다.

**→ 이 프로젝트에서 손목카메라의 효과를 오염 없이 잰 첫 비교다.**

## 2. 무엇을 돌렸나

- 팔 2개 × 지시문 2조건 = 4개 rollout, 각각 태스크 10개 × 20회 = 200회. 총 800회 전부 정상 완료
  (osmesa 렌더러 segfault·재시도 0회).
- 시뮬레이터 LIBERO-spatial, GPU 8/9만 사용. 성공/실패는 에피소드별 원장(episodes.jsonl)에서 직접 재집계.
- 체크포인트: `phase2_libero_large256_matchedbase.pt`(기준선, 오프라인 R² 0.655) /
  `phase2_libero_large256_wristpatch.pt`(W-A, R² 0.669).

### 사전 관문 G2 (돈 안 드는 오프라인 검사, 롤아웃 전에 통과)

"정책이 새 손목 토큰을 실제로 쓰는가?"를 검증 데이터에서 토큰을 0으로 지워보는 방식으로 확인:

| 조건 | 행동예측 R² | 해석 |
|---|---|---|
| 전부 정상 | 0.669 | (학습 때 기록과 소수 5자리까지 일치 — 재현 검증 통과) |
| 손목 패치 4토큰 지움 | 0.654 (−0.015) | **기준(−0.005)의 3배 하락 → 실제로 사용함** |
| 기존 손목 요약 토큰 지움 | 0.644 (−0.025) | 기존 토큰도 여전히 사용 |
| 둘 다 지움 | 0.563 (−0.106) | 두 손목 정보가 상호보완적 |

→ "토큰을 안 써서 롤아웃이 무의미"(F2 탈락 조건)는 **해당 없음**. 롤아웃 진행 정당화됨.

## 4. 언어 순응도 — 유일한 걱정거리

- 틀린 지시를 줬을 때 기준선은 11.5%만 성공(=지시를 잘 따름), W-A는 27.0% 성공.
  특히 t0(95%)·t2(85%)·t4(65%)는 **틀린 지시를 받고도 원래 태스크를 수행**했다.
- 해석: 손목 시야의 기하 정보가 강해지면서, 일부 태스크를 "지시문 없이 장면만 보고"
  풀 수 있게 된 것. 언어를 망가뜨렸다기보다 **언어 없이도 되는 경로가 생긴 것**이지만,
  언어 공동기준(정상−틀림 ≥ +70%p)에서는 +65.5%p로 5%p 미달.
- 사전등록 A4 유보 규칙: 20회 스크리닝의 이 지표는 잡음 ±10%p라, **65~75%p 구간은
  즉결 판정 금지 → 확정실험에서 재판정**. (즉결 탈락선인 +55%p 미만과는 거리가 멀다.)

## 5. 판정 및 다음 단계

| 관문 | 기준 | 결과 | 판정 |
|---|---|---|---|
| G2 토큰 사용 | ΔR² ≥ 0.005 | +0.015 | **통과** |
| F5 기준선 재현 | 85~88% | 87.0% | **통과** |
| 주 게이트: paired SR 우세 | 태스크 CI > 0 | [−2.5, +14.5] | 미달 (점추정 +5.5, 에피소드 하한 0.0) |
| 언어 공동기준 | c−w ≥ +70%p | +65.5%p | **유보** (A4 밴드 65~75) |
| F4 (유의 하락) | CI < 0 | 아님 | 발동 없음 |

**권고: W-A를 확정실험(50롤×3시드, 정상+틀림 지시)으로 승격.**
근거: ① 즉결 탈락 조건(F2/F3/F4/F5) 전부 미발동, ② 언어 유보 밴드는 규칙상 확정실험
재판정이 의무, ③ SR 신호가 양(+)이고 설계가 예측한 바로 그 태스크들에 집중 — 이 패턴이
우연일 가능성은 낮으나 20회 표본으로는 증명 불가. 확정실험(팔당 1,500회)이면
태스크-단위 검정도 판가름 난다. ④ 이번 세션 내내 "복잡도 추가는 전부 무익"이었는데,
이 팔은 **처음으로 양의 점추정 + 예측 적중 패턴**을 보인 후보다.

확정실험에서 볼 것: (a) +5.5%p가 유지되는가, (b) 언어 격차가 +70 선을 회복하는가
(t0/t2의 "지시 무시 성공"이 시드를 바꿔도 재현되는지), (c) t1(−15%p)이 잡음인지 실제 퇴행인지.

### 6.1 W-B (측정된 손목 변위 1토큰) — "W-A의 약한 복사본"

- 사전 관문 G0(오프라인): 그리퍼 예측 우위 **+0.085 ≥ +0.05 통과** (probe_g0_wrist_cell.json).
  학습 건강: N2 재척도 σ_ref/σ_Δ=4.94 정상 주입, 행동예측 R² 0.667 (W-A 0.669와 동급).
- 태스크별 이득이 W-A와 **같은 자리**에서 났다: t3 +15, t4(서랍) +30, t9(캐비닛) +20 —
  파지-지배 태스크 집중 예측 재적중. 쉬운 태스크는 −5%p 잡음.
- 그러나 평균 +4.5%p(91.5 vs 87.0)로 W-A(+5.5)보다 작고, 태스크-단위 CI [−2.0,+12.5]로
  0 포함(에피소드-단위 하한 0.0, p=0.96 — W-A와 동일한 "신호 있으나 검정력 부족" 패턴).
- 언어 격차 +63.0%p는 유보 밴드(65~75) **아래**라 언어 공동기준 미달(즉결 F3 선인
  +55/60에는 안 걸림). 틀린 지시에도 t0 85%·t2 90%·t4 90% 성공 — W-A에서 본
  "장면만 보고 푸는 경로"가 더 강하게 재현됐다.
- **판정: 널.** 사전등록 규칙상 승격 조건(CI>0 **그리고** c−w≥+70/유보밴드)을 둘 다
  못 넘었다. W-B는 W-A와 같은 태스크에서 같은 방향의 이득을 내지만 크기가 작고 언어
  훼손이 더 크다 = **W-A의 하위호환**. 결정트리의 "W-B 승리 → 병합팔 검토" 분기는
  열리지 않는다. 관측-측 손목 대표는 W-A 단독 유지.

### 6.2 W-C (dual-stream 표준화 재판) — 수리가 작동했는데도 폐루프는 못 넘었다

- 오프라인 수리는 전부 의도대로 작동했다: per-stream 표준화(dz_std 0.1346/0.0208)로
  두 스트림 align이 처음으로 **균형 수렴**(cos main +0.635 / wrist +0.643; 구판은 wrist가
  붕괴), 항별 gradnorm도 동급(0.56/0.40). 행동예측 R² 0.659(기준선 −0.01 밴드 통과).
- 사전등록 오프라인 게이트 **G2-C는 근소 미달**: ζ_wrist 절제 ΔR² **+0.251 < 0.27**
  (구판 0.179 → 표준화로 1.4배 강화됐으나 목표 1.5배엔 미달; oracle Δ는 0.101→0.361).
  미달 플래그를 달고 사전등록대로 롤아웃은 진행했다.
- 폐루프 결과는 게이트의 예측대로: 평균 **−4.5%p**(82.5 vs 87.0), CI [−11.5,+3.0]∋0 →
  통계적으론 "널"(유의 하락 F4까진 아님; 에피소드-단위 [−10.5,+1.5], p=0.06으로 하락
  경계). 하락이 파지 태스크가 아니라 t1 −20, t5 −20, t8 −15 등 **기준선이 잘 하던
  태스크**에서 났다 — 이득 자리(t4 +20)보다 잃은 자리가 많다.
- 언어 격차 +73.5%p는 유보 밴드(정상: 틀린 지시 9.0%만 성공) — 언어는 멀쩡하다.
  문제는 언어가 아니라 성공률 자체.
- **판정: 널(음의 점추정) — 타깃-측 손목(ζ_wrist 지도) 방향 종결.** "표준화 수리 후에도
  널"은 설계서 §5.3의 사전 예측 그대로다. G2-C 근소미달 → 폐루프 널의 순서도
  "오프라인 게이트가 폐루프를 과대예측하지 않았다"는 쪽으로 기록된다(게이트 신뢰 강화).

### 6.3 종합 권고 (결정트리 DESIGN_dualpolicy_dynamic_weighting_v1 §5 적용)

**분기 = "W-C 널 + W-A 승리(권고) 유지" → W-A 확정실험 단독 집중, ~45 GPU-h.**

- W-C 널 → joint 표준화 확증(~53 GPU-h)은 열리지 않음. 타깃-측 wrist는 4c 서사로 종결
  (frozen-기질에서 ζ_wrist 지도는 표준화 수리로도 폐루프 이득 전환 실패 — 강한 음성 행).
- W-B 널 → 병합팔 검토도 열리지 않음. 신규 학습 셀 추가 0.
- 확정실험(W-A, 50롤×3시드×2모드)에서 볼 것은 §5 그대로: (a) +5.5 유지, (b) c−w +70 회복
  여부 — W-B가 언어 훼손을 더 크게 재현한 만큼 (b)가 실질 관문이다, (c) t1 퇴행 여부.
